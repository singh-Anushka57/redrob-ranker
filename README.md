# Redrob Hackathon — Intelligent Candidate Ranker

## Approach

Two-phase system:
1. **Offline pre-computation** (no time limit): embed JD + all 100K candidates using BGE-small-en-v1.5, extract structured features, flag honeypots, pre-generate reasoning strings.
2. **Fast ranker** (≤5 min, CPU only, no network): load artifacts → cosine similarity → rule-based sub-scores → weighted fusion → top-100 CSV.

### Scoring model

| Component | Weight | Description |
|-----------|--------|-------------|
| Semantic similarity | 0.35 | BGE-small cosine(candidate_vec, jd_vec) |
| Skill match | 0.25 | Fraction of 29 hard-required IR/ML skills present |
| Experience | 0.20 | Years (target 4–12) × company type bonus |
| Title signal | 0.10 | Title keyword relevance (0–1) |
| Location | 0.10 | Preferred India cities / relocation willingness |
| Availability mult | ×factor | 23 behavioral signals → 0.35–1.35 multiplier |
| Disqualifier penalty | ×factor | Consulting-only / bad title / honeypot → 0.01–1.0 |

### Key design choices
- **No LLM at rank time** — reasonings are pre-generated and cached; rank.py just does a dict lookup.
- **Honeypot detection** — impossible tenure, expert-breadth impossibility, age anomalies.
- **JD intent over keywords** — title and company-type scores catch keyword stuffers regardless of skill list.

## Reproduction (step by step)

### Step 1: Pre-computation (run once, ~30–60 min total)

```bash
# 1a. Build JD embedding (~10 seconds)
python precompute/build_jd_embedding.py

# 1b. Parse and featurize all 100K candidates (~5 min)
python precompute/build_features.py candidates.jsonl

# 1c. Embed all candidates (~30–50 min on CPU)
python precompute/build_embeddings.py

# 1d. Flag honeypots (~3 min)
python precompute/flag_honeypots.py candidates.jsonl
```

### Step 2: Generate preliminary ranking + reasoning

```bash
# 2a. Run fast ranker in prelim mode (top-200 JSON)
python rank.py --candidates candidates.jsonl --prelim

# 2b. Generate reasoning strings for top-200 (offline, no API needed)
python precompute/generate_reasoning.py --candidates candidates.jsonl
```

### Step 3: Final submission (≤5 min, the timed step)

```bash
python rank.py --candidates candidates.jsonl --out team_xxx.csv
```

### Step 4: Validate

```bash
python validate_submission.py team_xxx.csv
```

Expected output: `Submission is valid.`

## Artifacts

After running precompute, `artifacts/` contains:

| File | Description |
|------|-------------|
| `jd_vec.npy` | JD embedding [1, 384] |
| `cand_vecs.npy` | Candidate embeddings [100000, 384] |
| `cand_ids.npy` | Candidate ID ordering |
| `features.parquet` | Structured features for all 100K |
| `honeypot_flags.json` | Flagged impossible profiles |
| `prelim_ranking.json` | Top-200 preliminary ranking |
| `reasoning_cache.json` | Pre-generated reasoning strings |

The `.npy` files are gitignored due to size. Regenerate with the precompute scripts.
